# Cycle de Versioning & Notes de Versions 📅🚀

Chaque projet au sein de la suite **mDevsLabs** suit un cycle de release strict pour garantir la stabilité et la transparence.

---

## 🏷️ Versioning Sémantique (SemVer)

Nous utilisons le standard SemVer `MAJOR.MINOR.PATCH` :
- **MAJOR** : Changements majeurs ou ruptures de compatibilité (ex: restructuration complète de l'API).
- **MINOR** : Ajout de fonctionnalités sans régression.
- **PATCH** : Corrections de bugs mineurs.

---

## ⚡ Canaux de Distribution

1. **Canal Stable** : Destiné à la production générale, testé de manière approfondie.
2. **Canal Canary** : Version expérimentale contenant les dernières fonctionnalités en cours de développement (ex: nouvelles capacités de vision sur `mAI`).
