# Contribuer à mProjects 🤝🌱

Nous encourageons la communauté à s'impliquer dans le développement des applications et modèles de la suite **mProjects**. Que ce soit pour signaler un bug, proposer une nouvelle fonctionnalité, ou améliorer la documentation, votre aide est précieuse !

---

## 🛠️ Processus de Développement

1. **Fork & Clone** : Créez un fork du dépôt officiel (par exemple, `mDevsLabs/mProjects`) et clonez-le sur votre machine locale.
2. **Créer une branche** : Utilisez un nom de branche descriptif, par exemple : `git checkout -b feature/nouveau-composant`.
3. **Coder avec soin** : Suivez les standards de code en vigueur (Prettier, ESLint).
4. **Vérifier** : Assurez-vous que le projet compile avec succès sans erreurs (`npm run build`).
5. **Pull Request** : Soumettez une PR vers la branche principale `main` avec une explication détaillée de vos modifications.

---

## 💬 Code de Conduite

Nous tenons à maintenir une communauté accueillante et respectueuse. Tous les contributeurs sont invités à faire preuve de bienveillance et de professionnalisme dans leurs échanges sur GitHub et sur notre serveur Discord.
